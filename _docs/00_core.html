Experiments on using Proposition Chunking and RAG indexing for semantic search

This is based on the 5 Levels Of Text Splitting notebook example for the “Agentic Text Splitting” method. It is based on the Dense X Retrieval: What Retrieval Granularity Should We Use? paper and has a prompt implementation in LangChain hub. Unfortunately, the LangChain hub is API walled, so I copied the prompt template to this notebook. We need to handle PDFs as a combination of text, image and tabular data for the purposes of chunking and indexing. LangChain blog has some interesting experiments on Benchmarking RAG on tables, Multi-modal RAG on slide decks and a set of notebooks Multi-modal eval: GPT-4 w/ multi-modal embeddings and multi-vector retriever. It also appears that the there is not a single PDF library that can handle text, tables and images for proficient extraction. Rather, pypdf2 is used for text extraction, camelot for table extraction and pypdfium2 for image extraction see Extracting Text from PDF Files with Python: A Comprehensive Guide. LangChain has a set of benchmarks LangChain Benchmarks that gives a deeper background on benchmarking. A tutorial for How to Use PDF Loaders in Langchain gives a good overview of the PDF extraction process. LangChain PDF Document Loaders Documentation gives details.
